# How to contribute?
Everyone is welcome to contribute to Syngine through bug reports (issues), testing on various platforms, writing examples, improving documentation, profiling and optimizing code, or promoting the project.

# Submitting bugs
For non-trivial bugs, please provide a reproducible case with code - this speeds up fixes.

# Contributing code
By contributing to Syngine, you agree to the MIT license terms.
Pull request checklist:
- Test your code on all platforms you have access to. List which platforms you did test on in the PR so we can test any remaining
- Keep PRs focused
- Use EditorConfig and Clang-Format to match the project's editor settings
